CORREÇÃO ERRO 500 - RENDER

Esta versão foi ajustada para evitar erro 500 causado por cookie antigo/sessão antiga ou falha de abertura do SQLite no Render.

Configurações no Render:
Build Command:
pip install -r requirements.txt

Start Command:
gunicorn app:app --bind 0.0.0.0:$PORT

Após subir esta versão:
1. Clique em Manual Deploy > Clear build cache & deploy.
2. Aguarde Deploy live.
3. Acesse /health e /health-db no final da URL.
4. Se /health-db mostrar OK, acesse a página principal.

Observação: esta versão ainda é de protótipo e usa SQLite. Para produção real com clientes, migrar para PostgreSQL.
